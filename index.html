설정

관리자 화면(편집·데이터)은 PIN으로 보호할 수 있어요. PIN을 정하면 ‘관리자’ 버튼을 누를 때 입력해야 들어갑니다. 라이브 화면은 누구나 볼 수 있어요. PIN은 브라우저에 저장되는 간단한 잠금이라, 강력한 보안은 아닙니다.

GitHub 게시 (사이트에 바로 저장)

이 값을 넣어두면, 관리자 화면에서 ‘사이트에 게시’ 한 번으로 현재 배치가 저장소의 데이터 파일로 올라가고 모든 방문자에게 반영돼요. HTML을 다시 업로드할 필요가 없습니다.

토큰은 저장소 콘텐츠 쓰기 권한만 가진 Fine-grained PAT를 권장해요. 토큰은 게시 파일에 포함되지 않고, 이 브라우저에만 저장됩니다. 공용 PC에서는 사용 후 지우세요.

데이터 관리

이름이 있는 좌석의 부서(실/팀)·담당업무·슬랙 ID를 한 곳에서 관리해요. 표에서 바로 고치거나, 엑셀 업로드 또는 CSV로 한꺼번에 올릴 수 있어요. (이름이 같으면 갱신, 없으면 새 좌석으로 추가)

엑셀로 추가/수정 엑셀 모듈을 못 불러왔어요(인터넷 필요). CSV 붙여넣기를 이용하세요.

저장 · 공유

사이트에서 관리하려면 이 버튼만 쓰면 돼요. 현재 배치를 저장소 데이터 파일(data.json)로 바로 올려, 모든 방문자에게 반영합니다. (설정에서 GitHub 저장소·토큰을 먼저 입력하세요.)

다른 화면의 배치를 통째로 가져오기 (벽·좌석·이름 전체)

① 배치가 보이는 화면에서 복사를 누르고 → ② 여기(예: 깃허브 관리자 페이지)에 붙여넣은 뒤 붙여넣기 불러오기. JSON은 물론 HTML 파일 내용 전체를 붙여넣어도 그 안의 배치를 자동으로 찾아 불러옵니다.
